Structure Overview

The repository follows a hierarchical structure:

root/
├── countries.json          # Main index of all countries
└── [country]/
    ├── operators.json      # List of operators in this country
    └── [operator]/
        ├── categories.json  # Package categories
        └── [category].json  # Actual packages

Package JSON Format

{
  "name": "Package Name",
  "price": "1738",
  "cost": 1738,
  "code": "*117*30#",
  "data": "25GB",
  "data_amount": 25,
  "unit": "GB",
  "validity": "30 Days",
  "duration_days": 30,
  "network": "Jazz",
  "active": true
}

How to Add a Country

  1. Add entry to countries.json
  2. Create [country]/operators.json
  3. Create operator folders with categories
  4. Add package JSON files in each category

Supported Categories

  • data - Internet/Data packages
  • social - Social media focused packages
  • voice - Call minutes packages
  • roaming - International roaming packages
  • combo - Mixed services packages

Tips for Maintenance

  • Keep prices updated regularly
  • Use consistent naming conventions
  • Include USSD codes for activation
  • Mark inactive packages with "active": false
  • Document special features and limitations
